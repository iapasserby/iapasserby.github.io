github https模式下切换账号

发表于 2017-02-07 | 分类于 git | | 阅读次数

前言

​ 偶然从实践者论坛上看到了如何用Hexo+Next+Github搭建博客的文章，参照文章一路过来也算顺利，因为之前一直用的另一个账号做的Demo，后面遇到用自己账号来正式搭建后部署 hexo d 显示权限有问题，因为之前用https登录过，所以系统默认会以此账号上传。

多账号切换问题

​ 因为我有两个github账号，所以在我用A账号登录过后再用B账号来部署就会出现权限问题。百度谷歌了很久看到的都是说通过SSH来实现多账号之间的切换 PUSH。关于Https方式的文章少之又少，而我想采用Https来实现多账号切换，所以始终不得其法。

​ 偶然从一个帖子上看到GIthub官网关于https登录的说明，在这个说明的底部放了一个相关文章的链接，讲得就是https情况下怎么去更新切换账号问题，豁然开朗，原来需要通过MAC下的钥匙串管理工具去删除旧的账号就行了，链接如下：https://help.github.com/articles/updating-credentials-from-the-osx-keychain/ 。

Hello 2017

发表于 2017-02-07 | | 阅读次数

​ 转眼已经是2017年过了农历春节后开始上班了，总感觉毕业参加工作还在不久前，时间真的过的很快。我对于时间的流逝是很有急迫感的，总觉得去年成长还不够快，没达到自己理想的状态，春节期间也一直在想自己是不是也应该写一些一年目标啥的来督促自己，所以决定以此开笔，以后有想记录的就多多记笔记，争取多记录写技术上遇到问题后解决的思路和过程，生活中的感悟等。

回顾2016

​ 2016年5月份参加工作，从此也从学生身份换成了社会人士身份，没有了之前的自由和闲适，同时也更多的时间花在了上班和加班上。但是，对于自己的成长和学习还是不能放弃的，白天工作干活，晚上加班回来也会看些公众号和书，时间珍贵，进度并不快，感受到了学习带来的快乐。收起了出去浪的心，竟然开始喜欢安静的呆着享受不多的时光。

工作

​ 作为新人参加工作一开始的感受总是不够感受的，我选择去适应和学习，先对环境和流程摸熟悉了再去思考，时间不够见解应该也是不到位的。作为iOS开发，之前一直以为是做APP上架，并没有想到自己会做SDK这一层，也很偶然竟然做起了SDK开发，当然也做了APP，只是深度方面来说还有点欠缺，因为产品是以提供SDK服务为主。反倒是SDK底层需要学习的东西比较多，我不知道这是好事还是坏事，虽然之前做C/C++开发接触的大部分都是底层。

​ 而对于一家以做云客服为主业的公司来说，当然移动端的需求就会变得不稳定，因为SDK稳定了就行了，而对于我来说我觉得自己还是个新人，所以紧凑的开发进度会使我更好的成长，这一点我的紧迫感也很强，总是希望通过不停的去实践来学习，这样会更明确一点，而漫无目的的学习会很不可持续性。

​ 所以在2017年，要更深入的去学习iOS开发，以及想顺便学一点JavaScript在移动端的应用。

阅读

​ 2016年总体来说应该是阅读量比较多的一年，在上班之前因为也已经完成了毕业答辩，所以有时间看了很多书，而参加工作只有也每天都会看一点，保持着一个缓慢的速度，一个月也能看1-2本书，所以对于这一点我还是比较满意的。

​ 而对于看书类型方面，我就比较杂了，从世界观到方法论，我都有看，还看了经济学数目，人物传记等，有时候感觉太杂了。所以2017年准备专一点，除了工作需要的技术书外大致方向往经商类和管理类多看点，文学类要偏少了，尤其是看着没那么有意思的书，要懂得取舍和快读。

展望2017

工作

​ 2017是完整工作的第一年，所以应该重在打基础和成长，基于去年的情况，今年要在iOS底层更深入，在做需求时要有独立担当，能想方案，想架构，懂优化，逐步往高层次走。同时，在空的时候学习前端技术，能有机会做一些需求最好，结合iOS和JavaScript应用，在这两端结合方面做一些研究。当然，有想法的时候尝试做一个自己的APP上架。

阅读

​ 人这一生，唯一不能停的就是学习，阅读。今年的目标就是更专，让财商类知识能够为自己带来额外的现金流。没有定具体的数目，因为床边的两叠书就已经足以看个一两年了。

​ 刚开始写的也比较简单，不过，只要是开始总是好的。
